# 🎂 Birthday Email Automator

Compose & send birthday emails · ICS/CSV import · Gift search · Heads-up reminders

---

## What's new in this version

- **Cloud sync via Supabase** — optional Postgres-backed persistent storage. Friends sync across devices automatically when configured. localStorage stays as the offline cache.
- **Hardened against XSS** — every user-rendered field (names, emails, notes, relations) is HTML-escaped before being inserted into the DOM
- **Input validation** — strict email format check, date format enforcement, control-character stripping, and length limits on all friend fields
- **Email subject CRLF protection** — strips line breaks and control characters from subjects to prevent header injection
- **Content Security Policy** — restrictive CSP meta tag locks down script, style, and connect sources
- **ICS & vCard import** — upload `.ics` or `.vcf` files from Google Calendar, Apple Contacts, Outlook, etc.
- **CSV import** — flexible column detection, multiple date formats, downloadable template
- **Settings tab** — configure EmailJS credentials in the UI (no rebuild needed)
- **CORS error handling** — clear diagnosis of network/quota/credential errors with "Open in mail app" fallback
- **Export CSV** — back up your friends list at any time

## Cloud sync setup (optional)

The app works fully offline with localStorage by default. To sync across devices and get true persistent storage, connect a free Supabase project:

1. Sign up at [supabase.com](https://supabase.com) (free tier, no credit card)
2. Create a new project
3. In the SQL editor, run:

```sql
create table friends (
  id bigint generated by default as identity primary key,
  name text unique not null,
  email text,
  birthday text not null,
  relation text,
  notes text,
  source text,
  created_at timestamp with time zone default now()
);
alter table friends enable row level security;
create policy "anon all" on friends for all using (true) with check (true);
```

4. Go to **Settings &rarr; API** in Supabase, copy your Project URL and the **anon public** key
5. In the app, open Settings &rarr; Cloud Sync, paste both values, and click **Save cloud sync**
6. Click **Sync now** to push existing friends to the cloud

After setup, every add/remove syncs in the background, and the app pulls remote friends on startup.

**Important:** Use the `anon public` key only -- never the `service_role` key. The anon key is safe for client-side use because Supabase enforces row-level security at the database level.

## Deploy to GitHub Pages

### 1. Add repository secrets *(optional — you can also set keys in the app's Settings tab)*

**Settings → Secrets and variables → Actions → New repository secret:**

| Secret name             | Value                   |
|-------------------------|-------------------------|
| `EMAILJS_PUBLIC_KEY`    | Your EmailJS public key |
| `EMAILJS_SERVICE_ID`    | e.g. `service_xxxxxxx`  |
| `EMAILJS_TEMPLATE_ID`   | e.g. `template_xxxxxxx` |

### 2. Enable GitHub Pages

**Settings → Pages → Source → GitHub Actions**

### 3. Push to main → auto-deploys

## EmailJS template setup

Create a template with these variables:
- `{{to_email}}` — recipient address
- `{{subject}}` — subject line
- `{{message}}` — email body
- `{{to_name}}` — recipient name (optional)

That's it. No other variables needed.

## Import formats

### ICS / vCard
- **Google Contacts:** Contacts → Export → vCard (.vcf)
- **Google Calendar:** Calendar Settings → Import & Export → Export (.ics)
- **Apple Contacts:** File → Export → Export vCard
- **Outlook:** File → Open & Export → Import/Export → Export to a file

### CSV columns (header row required, order flexible)
```
Name, Email, Birthday, Relationship, Notes
```
Date formats accepted: `YYYY-MM-DD`, `MM/DD/YYYY`, `DD-MM-YYYY`, `January 5 1990`
